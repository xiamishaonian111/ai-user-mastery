# AI 辅助编程专题 AI-assisted Programming

> 目标：掌握 AI 时代的编程工作流，让 AI 成为真正可靠的编程伙伴

---

## 一、Spec-driven Development（规格驱动开发）

**核心思路：** 先写清楚规格说明，AI 根据规格实现，用规格验证实现。

**为什么在 AI 时代更重要：**
AI 执行力强，但方向靠你定。Spec 越清楚，AI 实现偏差越小。

### Spec 应该包含什么

```markdown
# 功能规格：[功能名称]

## 目标
一句话说明这个功能要解决什么问题

## 输入
- 参数名：类型，说明，是否必填
- 边界条件：什么是合法输入，什么不是

## 输出
- 成功时返回什么
- 失败时返回什么（错误类型）

## 行为描述
- 正常流程：step by step
- 异常流程：各种错误情况怎么处理

## 不在范围内
- 明确列出这个功能不做什么

## 验收标准
- 怎么验证这个功能实现正确了
```

### 工作流

```
1. 你写 Spec
2. 让 AI review Spec（找漏洞、歧义、遗漏）
3. 修改 Spec 直到清晰
4. AI 根据 Spec 实现
5. 用验收标准验证
```

## 二、TDD with AI（测试驱动开发）

**核心：** 先写测试，AI 写代码让测试通过。

**为什么有效：**
- 测试是 Spec 的可执行版本，AI 有明确目标
- 测试通过 = 实现正确，不需要人工猜测
- 防止 AI 写出"看起来对但有 bug"的代码

### 工作流

```
Step 1：你（或 AI）写测试用例
  → 正常情况
  → 边界情况
  → 错误情况

Step 2：确认测试用例覆盖了你的需求

Step 3：让 AI 写代码让测试通过

Step 4：运行测试，如果没过，让 AI 修复

Step 5：重构（可选），确保测试仍然通过
```

**关键：** 测试用例由你来确认，不能完全交给 AI 写——它可能写出只覆盖它实现方式的测试。

## 三、大型代码库导航

**挑战：** 代码库太大，AI 的 context window 装不下，怎么让 AI 有效工作？

### 关键策略

**1. CLAUDE.md 写清楚项目结构**
```markdown
## 项目结构
- 核心业务逻辑：/src/core/
- API 层：/src/api/
- 数据库模型：/src/models/
- 测试：/tests/

## 关键文件
- 入口：/src/main.py
- 配置：/config/settings.py
- 路由：/src/api/routes.py
```

**2. 用 index 文件做模块导航**
每个主要模块放一个 README，说明这个模块是什么、主要文件有哪些。

**3. 任务粒度要小**
不要让 AI 一次理解整个代码库。每次只给它相关的文件。

**4. 让 AI 先探索再实现**
```
先告诉我这个代码库里，
[功能X] 相关的文件有哪些，
理清楚以后再开始实现。
```

## 四、Code Review 工作流

**系统性让 AI review 代码，不只是"帮我看看这段代码"。**

### Review 维度清单

```
[Role] 你是一个资深工程师，进行严格的 code review。

请从以下维度逐一检查，每个维度都要给出结论：

1. 正确性：逻辑是否正确，有没有 bug
2. 边界情况：是否处理了所有边界情况
3. 安全性：有没有安全漏洞（注入、越权等）
4. 性能：有没有明显的性能问题
5. 可读性：代码是否清晰易懂
6. 可维护性：修改这段代码容易吗

每个问题标注严重程度：BLOCKING / WARNING / SUGGESTION
```

## 五、安全重构工作流

**挑战：** 重构时如何确保不破坏现有功能？

```
Step 1：让 AI 先理解现有代码的行为
  → 输出：这段代码做了什么，有哪些隐含行为

Step 2：确认 AI 的理解是否正确

Step 3：让 AI 写测试覆盖现有行为（重构前）

Step 4：重构

Step 5：跑测试，确认行为没有改变
```

**关键：** Step 3 是保险——有了测试，重构出问题立刻知道。

## 六、Debugging with AI

**不要只把报错贴给 AI，要给它足够的上下文。**

### Debug 信息模板

```
Bug 描述：[现象是什么]

期望行为：[应该发生什么]
实际行为：[实际发生了什么]

错误信息：
[完整的 error message 和 stack trace]

相关代码：
[只贴相关的代码片段，不是整个文件]

已经尝试过：
[你已经试过什么方法]
```
