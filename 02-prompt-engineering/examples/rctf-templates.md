# RCTF 模板库

可以直接复制使用，根据情况填空。

---

## 代码类

```
[Role] 你是一名资深软件工程师，专注于代码质量和可维护性。

[Context] 这是一个 Python 后端服务的核心模块，
会被多个团队共用，需要长期维护。

[Task] Review 以下代码，找出：
1. 潜在的 bug 或边界情况
2. 性能问题
3. 可读性和可维护性问题

每个问题注明严重程度：BLOCKING / WARNING / SUGGESTION

[Format] 按严重程度分组列出，每条格式：
- 行号：X
- 问题：...
- 建议：...

代码：
[粘贴代码]
```

---

## 文档写作类

```
[Role] 你是一名技术写作专家，擅长把复杂概念用清晰的语言表达。

[Context] 目标读者是非技术背景的产品经理，
他们需要理解这个功能但不需要知道实现细节。

[Task] 把以下技术说明改写为产品说明文档。

[Format] 
- 标题简洁
- 每段不超过 3 句话
- 用"你"而不是"用户"
- 不使用技术术语，如有必要用括号简单解释
- 最后加一个"常见问题"部分，2-3 个问题

原文：
[粘贴原文]
```

---

## 分析决策类

```
[Role] 你是一名有经验的产品经理和业务分析师。

[Context] 我们是一个 B2B SaaS 公司，月活 5000，
主要客户是中小企业。目前在考虑是否要做移动端 App。

[Task] 分析这个决策的利弊。请先列出需要考虑的因素，
再逐一分析，最后给出你的建议和理由。

[Format]
1. 支持做的理由（3-5 条）
2. 反对做的理由（3-5 条）
3. 关键前提条件（做这个决定前需要知道什么）
4. 建议和理由（1 段，100 字以内）
```

---

## 总结提炼类

```
[Task] 总结以下内容的核心要点。

要求：
- 3-5 个要点，每点一句话
- 不要照抄原文，用自己的话概括
- 不要加任何前置说明，直接给要点列表
- 如果有数据，保留关键数字

内容：
[粘贴内容]
```
